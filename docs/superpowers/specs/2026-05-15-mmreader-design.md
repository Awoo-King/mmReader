## 1. 目标与范围

mmReader 是一个面向 macOS 的极简透明阅读器，用于在工作场景下进行低干扰阅读。当前迭代优先级为“稳定正确优先”，先确保核心行为一致、可恢复、可回归验证，再推进交付打包。

## 2. 分层范围（P0 / P1 / P2）

### 2.1 P0（稳定正确，必须先达成）

**目标：** 在 Xcode Run 场景下，核心阅读链路长期可用且状态一致。

**包含：**
- 无边框窗口（`borderless`）与纯文本阅读视图。
- 主阅读区不显示页码和进度条。
- 页码与阅读进度仅在工具栏显示。
- 工具栏支持 `⌘B` 显示/隐藏。
- 支持打开 `.txt` 与 `.md` 文件（含拖拽）。
- 快捷键仅在应用激活时生效（不做全局快捷键）。
- 配置持久化与恢复：字体、透明度、行数、窗口几何、置顶、全屏、上次文件、上次页码。
- 文件读取失败、格式不支持、配置损坏等异常不破坏当前阅读状态。

**不包含：**
- `.app` 打包交付。
- 点击穿透。
- Markdown 富文本渲染（首版按纯文本处理）。
- 全局系统级热键。

## 3. 架构设计

采用 Swift + AppKit 多文件极简架构：

1. `AppDelegate.swift`
   - 应用入口。
   - 菜单和应用内快捷键注册。
   - 生命周期事件触发配置保存。

2. `ReaderWindowController.swift`
   - 创建并管理 `borderless` 主窗口。
   - 处理窗口拖动、置顶、全屏、工具栏显隐。
   - 处理窗口移动/尺寸变更并触发持久化。

3. `ReaderView.swift`
   - 仅负责文本绘制。
   - 不绘制任何状态信息。
   - 最小内边距与 4pt 行间距。

4. `ToolbarView.swift`
   - 显示页码（`current/total`）与百分比。
   - 提供常用操作入口（打开、翻页、字体、透明度、置顶、全屏、工具栏开关）。

5. `ReaderEngine.swift`
   - 文件加载（TXT/MD）。
   - 文本规范化。
   - 分页计算与页索引维护。
   - 窗口/字体变化时重分页及进度映射。

6. `ConfigStore.swift`
   - 读写 `~/.mmreader/config.json`。
   - 提供默认值与损坏配置回退。

## 4. 关键技术约束（稳定正确优先）

### 4.1 分页一致性约束

`ReaderEngine` 维护两套定位信息：
- 硬定位：`pageIndex`
- 软定位：`anchor`（页首字符全局偏移，或等价的段落定位锚点）

重分页（字体、窗口尺寸、每页行数变化）时执行：
1. 先依据 `anchor` 在新分页结果中定位最接近页。
2. 再回写新的 `pageIndex`。
3. 若 `anchor` 越界（如文件变短），钳制到最后一页。

验收口径：重分页后保持语义连续，而非机械保留同页号。

### 4.2 状态写入模型

配置写入采用单入口策略：
- 统一由 `persistConfig()` 写配置，不允许多处直接写文件。
- 高频事件（窗口拖动/缩放）做短去抖（150–300ms）。
- 写入失败仅记录并提示，不影响当前会话。
- 写入失败时不清空内存状态。

### 4.3 启动恢复顺序

固定顺序，避免状态覆盖：
1. 读取配置；若损坏则回退默认并覆盖写回。
2. 应用窗口几何与显示参数。
3. 尝试恢复上次文件。
4. 文件加载成功后恢复阅读位置（`anchor/pageIndex`）。
5. 若上次文件不存在或不可读，跳过文件恢复但保证窗口可用。

### 4.4 文件加载原子切换

打开新文件采用两阶段提交：
1. 在临时模型中完成读取、规范化、分页。
2. 全部成功后一次性替换当前阅读状态。
3. 任一步失败则保留旧文档与旧页，不做部分覆盖。

### 4.5 明确边界（P0 不做）

- 不处理跨设备同步与并发编辑冲突。
- 不做大型文件性能专项优化（当前迭代只保证正确性）。
- 不对 Markdown 富文本渲染一致性做承诺。

## 5. 关键交互与数据流

### 5.1 启动

1. 读取配置；若损坏则回退默认并覆盖写回。
2. 恢复窗口位置、大小、置顶、全屏与显示参数。
3. 若 `lastFilePath` 存在且可读，加载文件并恢复阅读位置。
4. 启动完成后显示主窗口。

### 5.2 打开文件

1. 菜单、快捷键、拖拽任一路径进入加载流程。
2. 校验扩展名仅允许 `.txt`、`.md`。
3. 临时完成读取与分页，成功后原子替换。
4. `ReaderView` 渲染当前页。
5. `ToolbarView` 更新页码和百分比。
6. 更新并持久化 `lastFilePath` 与阅读位置。

### 5.3 状态更新

- 字体大小调整
- 背景透明度调整
- 文本透明度调整
- 每页行数调整
- 置顶切换
- 全屏切换
- 窗口移动与缩放

以上变更统一走 `persistConfig()` 保存。

### 5.4 工具栏显隐

- `⌘B` 切换工具栏。
- 隐藏后阅读区保持纯文本，不显示任何附加 UI。

## 6. 配置模型

配置文件路径：`~/.mmreader/config.json`

字段：
- `fontSize: Double`
- `bgAlpha: Double`
- `textAlpha: Double`
- `linesPerPage: Int`
- `windowX: Double`
- `windowY: Double`
- `windowWidth: Double`
- `windowHeight: Double`
- `isPinned: Bool`
- `isFullscreen: Bool`
- `lastFilePath: String?`
- `lastPageIndex: Int`
- `lastAnchor: Int?`（新增：用于重分页连续性恢复）

## 7. 渲染规范

- 默认系统字体。
- 行间距固定为 4pt。
- 最小边距策略（贴边阅读并保留最小可读留白）。
- 主视图绝不绘制页码、百分比、进度线。

## 8. 错误处理

- 不支持格式：提示“仅支持 TXT/MD”。
- 文件读取失败：提示并保持当前阅读状态不被清空。
- 上次文件不存在：忽略自动恢复文件，应用正常启动。
- 配置损坏：回退默认配置并保存。
- 配置写入失败：提示并记录，当前阅读会话继续。

## 9. 验收与回归矩阵

### 9.1 P0 验收矩阵（必须全过）

1. 启动恢复
   - 前置：已有有效配置 + 上次文件存在。
   - 期望：窗口几何、透明度、字体、置顶/全屏、页码均恢复。

2. 配置损坏恢复
   - 前置：写入非法 `config.json`。
   - 期望：应用可启动，自动回退默认并重写配置。

3. 文件打开原子性
   - 前置：先打开可读文件 A，再尝试打开不可读或不支持文件 B。
   - 期望：B 失败后仍停留在 A 当前页内容。

4. 重分页连续性
   - 前置：在长文中部阅读，调整字体与窗口尺寸。
   - 期望：重分页后仍在原语义附近。

5. 工具栏与主阅读区职责隔离
   - 前置：显示/隐藏工具栏并翻页。
   - 期望：页码与百分比仅在工具栏显示。

6. 快捷键作用域
   - 前置：应用激活与非激活状态分别测试。
   - 期望：仅激活时响应快捷键。

### 9.2 P1 验收矩阵（交付可用）

1. 打包可复现：按固定步骤生成 `mmReader.app`。
2. 双击启动：Finder 双击可启动。
3. 行为一致：打包版与 Xcode Run 的 P0 行为一致。
4. 重启回归：打包版关闭重开后状态恢复一致。

### 9.3 最低回归清单（每次改动后）

- 打开 TXT、打开 MD、拖拽打开。
- 翻页 + 调字体 + 调窗口大小（检查连续性）。
- 工具栏显隐切换。
- 关闭重开恢复。
- 故障注入：不支持格式 / 不可读文件 / 坏配置。

## 10. 迭代边界

当前设计仅覆盖单机本地阅读器稳定化与打包交付路径，不扩展到协同、同步、多格式富文本引擎。
